  1. What is the Document Object Model (DOM)?
    1. The Document Object Model is an in-memory data structure that represents the browser's HTML web document. It is, in essence, a bridge that connects the HTML and JavaScript, allowing programmers to turn HTML elements into manipulatable elements in JS. This is done by linking the HTML element to a JS variable using document.querySelector(HTML tag, class/id name). By storing a reference to the HTML tags as JS variables, we enable the JS code to access and alter the HTML.
  2. What is the difference between HTML and the DOM?
    1. The HTML is what the web-browser displays to the user, whereas the DOM is the bridge connecting the visible HTML with the invisible JavaScript functions. HTML is a marionette puppet and the DOM is the strings. When JS calls on a variable in the DOM, it effectively pulls on the string and the user sees the puppet (or HTML) move. Just as the puppet is not the same as the strings but is manipulated by the use of strings, so the HTML is not the same as the DOM but is manipulated by the DOM linking it to the JS.
  3. What are the main types of DOM nodes?
    1. The DOM can best be thought of as a tree, each part of which represents a separate node, which is accessible to JS through the DOM. The roots of the DOM are present in the name: Document. The document is written in HTML, so this is the trunk. Next, the tree splits into its two main branches: head and body. The headhas its own branches, which include the title and metadata. The body is split in various ways, first by header nodes (1-6). Headers can include lists (ordered and unordered) which represent a single node. Likewise, each of the list items represents another node.
  4. How can you traverse the DOM tree and access specific elements or nodes?
    1. You can traverse the tree and grab specific elements using the document.querySelector() method. You must be specific in how you grab particular elements, as the DOM will access the first element it finds in the code according to the tags you've called. For instance, if both h1 and h2 have p tags, and you wanted to access the paragraph in the h2 specifically, you would need to list the arm of the tree that this tag belongs to: document.querySelector(h2 < p)
      This would avoid the issue of the DOM grabbing the wrong p tag.

  7. What is the difference between a "live" NodeList and a "static" NodeList? When would you use each?
    1. A live node list will recognize when a new element has been appended to the HTML from the DOM and will render the appended elements to the html, such that they can be returned to the console or returned by a function. A static NodeList will not do this, thus appended items will not be referenced when called upon. I suppose you would use a static nodelist when you want to retain the identity of the original listed elements, whereas a live nodelist would be best when you want the appended items to be included with the earlier ones when being called upon.

    1. Event delegation allows you to write DRYer code by taking as the event handler some kind of parent element, which can pass the function onto each of the child elements, rather than writing the function over and over for each of the child elements.
